# Vision and mission

Simulation is central to modern robotics research: it trains policies, generates data, and turns computation into capability. Yet researchers have long been held back by two problems with the simulators available to them — **usability** and **opacity**. GPU-accelerated simulators often carry steep learning curves, with data-centric abstractions, large APIs, and complex software stacks that are hard for newcomers to master. Others are closed source, so researchers cannot inspect the physics, adapt it to real-world observations, or improve it.

Genesis World exists to close that gap. Our vision is a transparent, welcoming ecosystem where researchers from physics simulation and robotics build a high-efficiency, physically and visually realistic virtual world together — and where the algorithms the computer-graphics community continually produces for simulation and rendering are brought into robotics rather than left on the shelf.

We have worked to make Genesis World as easy to use as possible, with intuitive APIs and a clean, lightweight design. It is early, and a small team cannot get everything right on a first release, so we welcome contributors from any background to help improve it for the whole community.

If you have feedback while using Genesis World — a bug report, a feature request, or an idea for a better API — please open an issue or a pull request on [GitHub](https://github.com/Genesis-Embodied-AI/genesis-world). We would love to hear from you.
